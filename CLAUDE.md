## 1. 절대 하지 말 것

### 문서를 마음대로 고치지 않는다

문서 수정은 **채팅(claude.ai)에서만** 한다. 노션과 레포를 양쪽으로 관리하므로,
Claude Code가 임의로 문서를 고치면 노션이 조용히 뒤처진다.

**예외** — 사용자가 준 지시문에 문서 수정이 명시적으로 포함된 경우에만 고친다.

### 미확정 사항을 코딩하지 않는다

§3에 걸리는 게 나오면 멈추고 물어본다. 추측해서 만들지 않는다.
**확정과 미확정을 한 문장에 섞지 않는다.** 애매하면 미확정이다.

### 이지선다를 양쪽 다 만들지 않는다

선택지가 둘이면 둘 다 코딩하지 말고 **사용자 선택을 기다린다.**

### 기능을 제안하지 않는다

**스코프 방어가 우선이다.** 대학 팀 프로젝트 규모를 넘기지 않는다.
"이것도 있으면 좋을 텐데"는 §3에 올리는 것이지 만드는 게 아니다.

## 2. 비협상 (코드로 어겨선 안 되는 것)

로드맵 §1 원칙에서 온 것들. 위반을 발견하면 **고치기 전에 보고**한다.

1. **EXIF(GPS 포함)는 무조건 제거한다.** `lib/image.js`의 캔버스 재인코딩 경로를
   우회하는 업로드를 만들지 않는다
2. **목격 좌표는 저장 직전 50m 격자로 반올림한다.** `lib/geo.js`의 `snapToGrid`를
   통과하지 않는 `sightings` INSERT는 없어야 한다. 정밀 좌표는 DB에 존재하지 않는다
3. **목격 노출은 1시간 지연.** 프론트가 숨기는 게 아니라 DB 뷰(`cats_guest`/`cats_full`)가
   안 준다. 지연을 우회하는 조회를 만들지 않는다
4. **실시간 위치·경로 안내·위치 기반 알림 금지.** "찾아가게" 만들지 말고
   "우연히 만났을 때 알아보게" 만든다
5. **게스트에게 열리는 건 플로팅 카드 하나뿐.** 컬럼 차단은 DB 뷰가, 화면 차단은 UI가 한다

## 3. 코드 작성 방식

- **부분 스니펫 말고 완전한 파일**로 준다. 주석은 인라인으로
- 기존 함수 시그니처를 그대로 쓴다. **중복 함수를 만들지 않는다**
- 디자인 토큰만 쓴다 — `src/styles/app.css`의 `--token-*`.
  프로토타입 색(`#e51d53` 등)·폰트(`Gowun Batang`)를 새로 넣지 않는다
- 아이콘은 `lucide-react` outline 전용
- 파일 여러 개를 건드리면 **무엇을 왜 바꿨는지** 마지막에 요약한다
- 설명은 최소로. **어디를 어떻게 고쳤는지만.** 부가 설명은 요청받을 때만

### 디자인 토큰

```
paper #FBF7EF · sage #D7E0CC · deep green #3A5A40 · amber #C0842A · ink #2A2A24
제목·고양이 이름 Gaegu / 본문 Gowun Dodum / 폴백 Noto Sans KR
```
