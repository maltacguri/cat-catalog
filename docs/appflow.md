## 1. 진입 · 인증

```
start
 └ 스플래쉬 페이지
    └ 토큰 O / X ?
       ├ 있음 → 메인지도 페이지
       └ 없음 → 로그인 화면
                └ 계정이 있는가?
                   ├ 있음 → 로그인 페이지 → 메인화면
                   └ 없음 → 회원가입 할 것인가?
                            ├ 함 → 어떤 방식으로?
                            │      ├ 이메일 → 이메일 회원가입 → 비밀번호 설정 ┐
                            │      └ 카카오톡 → 카카오톡 로그인 절차 시작 ────┤
                            │                                                 ▼
                            │      고양이 관련 퀴즈 또는 당부사항 팝업
                            │       └ 고양이를 보호하겠다는 서약서 작성
                            │          └ 회원가입 완료
                            └ 안 함 → 게스트로 이용하기 팝업
                                      └ 당부사항 · 서약서 축약본 팝업
                                         └ 메인지도 페이지
```

**메모**

- 학교 이메일 사용 X
- 이메일 인증은 OTP 방식, 가입 시 1회
- **카카오 프로필(닉네임·사진)은 도입하지 않는다** — 로드맵 §2.1 (2026-07-20 재확인)
- 네이버 로그인 없음
- 회원가입 완료 플래그는 `profiles.onboarded_at`
- 서약서는 개인정보 동의서처럼 하되 코믹하게. 고양이 발바닥 도장 같은 거 찍으면 귀여울 듯

## 3. 고양이 프로필

지도의 고양이 아이콘 클릭 시 진입.

**기본 화면(지도)에서 보이는 고양이 정보 — CatFloatingCard, 게스트 열람 가능**
사진 · 이름 · **특이사항** · 밥 언제 먹었는지
**성별은 여기 없다.** 성별은 상세 프로필에만 (로드맵 §2.2 / §2.7-1, 2026-07-21 확정)

**프로필 안 내용**

- 사진
- 최근 밥 · 물 · 간식 등 언제 받았는지
- 특이사항
- 중성화 여부 등
- 남들이 언제 밥 줬는지 확인 가능
- 다른 사람이 해당 고양이에 대해서 올린 사진들도 확인 가능

**분기**

```
고양이 프로필
 ├ 고양이 정보 업데이트 할 것인가?
 │   └ Yes → 수정 버튼 클릭 → 정보 업데이트
 └ 고양이에게 밥을 줬는가?
     └ Yes → 밥주기 버튼 클릭 → 밥주기 모달 → 밥주기 프로필 업데이트
```

**메모** — 최근에 밥 준 기록 확인해서 너무 최근이면 경고 팝업 띄우기

## 4. 고양이 등록

지도 우측 하단 + 버튼에서 시작.

```
고양이 등록
 └ 등록하고자 하는 고양이가 이미 등록됐는지 확인
   (기준: 1. 위치  2. 색)
    ├ 주변에 등록된 고양이가 있는 경우
    │   └ "이 고양이가 맞습니까?" 화면 출력
    │      └ 등록돼 있으면 → 등록된 고양이의 프로필로 이동
    └ 등록 없는 경우
        └ 주소검색 / 드래그를 통해서 마크 찍기
           └ 상세정보 입력 · 고양이 등록 모달 오픈
              └ 고양이 등록 완료 → 메인페이지(지도)로 이동
```

**메모**

- 사진 없으면 **색 기준 기본 제공 일러스트 중 선택** (AI 생성 이미지 아님), 한 장이면 바로 대표사진
- **등록은 로그인한 사용자 누구나 가능하다** (로드맵 §2.8-12). 게스트는 불가

## 5. 주변 고양이 보기

```
주변 고양이 보기 → (위치 기반 선제시) → 주변 고양이 도감
```

## 7. 게스트(비로그인) 권한

- 지도에서 고양이 확인 가능 (기본 화면에서)
- 상세 프로필 확인 불가. **대표사진 한 장 · 이름 · 특이사항 · 최근 밥**까지만 확인 가능
- 다른 사용자가 올린 사진 접근 불가
- 마이캣 · 집사프로필 · 주변 고양이 보기 불가
  (맵에서만 확인 가능하게 → 다운로드 및 회원가입을 요구하는 요인)
- **하단 바 4탭은 게스트에게도 전부 보인다. 잠금 상태로 노출**하고, 누르면 화면 전환 없이 로그인 유도를 띄운다 (로드맵 §2.2)

> 그림 왼쪽의 "게스트에게 어디까지 보여줄지 논의 및 결정 필요" 메모는 이 §7로 해소됐다. 다시 묻지 않는다.

> **차단 지점 구분** (로드맵 §2.8-11, 2026-07-21 (4) 정정)
> - **컬럼 차단 = DB 뷰.** anon은 `cats` 테이블을 직접 읽지 않고 `cats_guest` 뷰만 읽는다. 성별·중성화는 애초에 내려오지 않는다
> - **화면 차단 = UI.** 도감 리스트·검색·마이캣·집사프로필은 UI 레벨에서 막는다
> - **목격 1시간 지연도 뷰가 적용한다** (로드맵 §2.3). `cats_guest` / `cats_full` 둘 다
>
> *이 문단은 원래 "화면 레벨 제한이고 anon은 `cats`를 직접 읽을 수 있다"로 적혀 있었다. 로드맵 (4) 정정으로 폐기됨.*

## 8. 로드맵 v2와 어긋나던 지점 — 처리 결과 (2026-07-21)

**7개 전부 닫혔다. 앱플로우와 로드맵 사이에 남은 충돌 없음.**

- **A. 고양이 등록 권한** — ✅ **로그인한 사용자 누구나로 확정** (로드맵 §2.8-12). "동아리원·관리자만"은 폐기
- **B. 카카오 닉네임·프로필 사진** — ✅ 저장·표시 안 함 (로드맵 §2.1 유지)
- **C. 사진 없을 때 AI 일러스트** — ✅ **색 기준 기본 제공 일러스트 중 선택**으로 확정
- **D. 주변 고양이 보기** — ✅ v1 포함. 원칙 1 문구를 좁혀 충돌 해소
- **E. 밥·물·간식 카운터** — ✅ v1 포함. **밥/간식/물 3종 확정** (로드맵 §2.8-15)
- **F. 가입 퀴즈 · 서약서** — ✅ v1 포함
- **G. 지도 기본 화면의 성별 표시** — ✅ **성별은 상세 프로필에만.** 카드에서 제거

**팔로우 / 좋아요 / 피드는 v2 이후다.** v1에서는 밥·간식 카운팅까지만 간다.

다음은 당시 기록이다.

| # | 앱플로우 |
|---|---|
| A | §4 — 메인지도 + 버튼으로 누구나 고양이 등록 |
| B | §2.1 카카오 닉네임·프로필 사진은 저장도 표시도 안 함 |
| C | §4 — 사진 없으면 AI 일러스트, 색만 가지고 기본 제공된 이미지 중 선택하는 방식 |
| D | §5 — 주변 고양이 보기(위치 기반 검색) |
| E | §6 — 밥·물·간식 준 횟수 카운터 |
| F | §1 — 가입 퀴즈 · 서약서 |

§3은 지도 기본 화면 정보에 **성별**을 넣었는데, §7 게스트 권한에는 성별이 없고 **특이사항**이 있다. → **성별은 상세프로필에만 제공**

§2.6 팔로우/좋아요/피드 같은 소셜 기능은 v2 이후 · §3-6. 반드시 도입할 것이나 v1에서는 밥 및 간식 카운팅에 초점.
